Oblea is a B2B platform that becomes a company's **AI layer**, starting with automatic quote generation. A customer sends a quote request to a company's sales inbox; Oblea reads it, classifies it, extracts the products, matches them against the company's catalog, and produces a quotation the sales person only has to review — not build from scratch.

This is the reference for **`oblea-backend`**, the FastAPI service that is the source of truth for all data (companies, catalogs, customers, messages and quotations) and the only service that writes to the database.

## The three services

`oblea-backend` never works alone. It sits between the frontend the sales team uses, the `inbound.new` email infrastructure, and the stateless AI agent.

<CardGroup cols={3}>
  <Card title="oblea-frontend" icon="window">
    Next.js platform. Reads and writes through this API with the signed-in user's **Supabase JWT**.
  </Card>
  <Card title="oblea-backend" icon="server">
    This service. FastAPI + Supabase (Postgres). Owns every write to the database.
  </Card>
  <Card title="oblea-agent" icon="robot">
    Stateless LangGraph agent. Extracts and matches items, then hands the finished draft **back** to this API.
  </Card>
</CardGroup>

## Two kinds of endpoints

The API has two audiences, and they authenticate differently.

<CardGroup cols={2}>
  <Card title="Platform endpoints" icon="user-lock">
    Called by the frontend on behalf of a signed-in user. Require an `Authorization: Bearer <supabase-jwt>` header and resolve to the one company that user owns.

    `/integrations` · `/inbox` · `/quotations`
  </Card>
  <Card title="Internal endpoints" icon="right-left">
    Called machine-to-machine by `inbound.new` and `oblea-agent`. **No user auth today** — they trust the caller.

    `POST /webhooks/receive_message` · `POST /quotations/{id}`
  </Card>
</CardGroup>

<Note>
  Every platform endpoint is scoped to a single company. The company is resolved from the authenticated user (`company.user_id`), never taken from the request body — so one user can only ever read or write their own organization's data.
</Note>
