<div className="text-sm font-mono">
  <span className="font-semibold text-blue-500">GET</span> `/quotations/{quotation_id}`
</div>

Returns a single quotation **enriched for the review editor** — not the raw database row. It stitches together the seller `organization`, the customer `client`, the original `source_message`, the proposed reply, and every line `item` (merged with its catalog data and per-line suggested alternatives) in one response, so the frontend can render the whole editor from a single call.

## Authorization

Requires a platform JWT. See [Authentication](/authentication).

```
Authorization: Bearer <supabase-access-token>
```

The quotation is scoped to the caller's company — a quotation belonging to another company resolves as **not found**.

## Path parameters

<ParamField path="quotation_id" type="string" required>
  The id of the quotation to fetch.
</ParamField>

  The quotation's unique id.
</ResponseField>

<ResponseField name="name" type="string | null">
  Short AI-generated title for the quotation.
</ResponseField>

<ResponseField name="quotation_number" type="string | null">
  Human-facing quotation number, once assigned.

<ResponseField name="status" type="string">
  Lifecycle status — `PROCESSING`, `READY TO REVIEW`, or `SENT`. See [The quotation lifecycle](/concepts/quotation-lifecycle).
</ResponseField>

<ResponseField name="sales_person" type="string | null">
  The sales person assigned to the quotation. Defaults to the company email.
</ResponseField>

<ResponseField name="date" type="string | null">
  The quotation date.
</ResponseField>

<ResponseField name="validity" type="string | null">
  How long the quotation is valid.
</ResponseField>

<ResponseField name="iva_rate" type="number">
  Tax rate snapshot applied to the quotation (e.g. `0.16`).
</ResponseField>

<ResponseField name="notes" type="string | null">
  Free-text notes attached to the quotation.
</ResponseField>

<ResponseField name="organization" type="object">
  The seller company header — the organization issuing the quotation.

  <Expandable title="organization">

    <ResponseField name="name" type="string | null">Company name.</ResponseField>
    <ResponseField name="logo" type="string | null">Logo URL.</ResponseField>
    <ResponseField name="address" type="string | null">Company address.</ResponseField>
    <ResponseField name="phone" type="string | null">Company phone.</ResponseField>
    <ResponseField name="email" type="string | null">Company email.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="client" type="object">
  The customer requesting the quote. Every field is `null` until the agent fills the draft.

  <Expandable title="client">
    <ResponseField name="id" type="string | null">The client id.</ResponseField>
    <ResponseField name="name" type="string | null">Contact name.</ResponseField>
    <ResponseField name="organization" type="string | null">The customer's company name.</ResponseField>
    <ResponseField name="address" type="string | null">Client address.</ResponseField>
    <ResponseField name="email" type="string | null">Client email.</ResponseField>
    <ResponseField name="phone" type="string | null">Client phone.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="send_message" type="object | null">
  The proposed reply drafted by the agent. `null` if none was generated.

  <Expandable title="send_message">
    <ResponseField name="subject" type="string">Subject line of the proposed reply.</ResponseField>
    <ResponseField name="body" type="string">Body of the proposed reply.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="source_message" type="object | null">
  The original inbound email that triggered the quotation. `null` if the quotation was not sourced from a message.

  <Expandable title="source_message">
    <ResponseField name="subject" type="string | null">The email subject.</ResponseField>
    <ResponseField name="text" type="string | null">The email body text.</ResponseField>
    <ResponseField name="from_address" type="string | null">The sender's address.</ResponseField>
    <ResponseField name="channel" type="string | null">The channel it arrived on (e.g. `email`).</ResponseField>
    <ResponseField name="sent_at" type="string | null">The email's own date header.</ResponseField>
    <ResponseField name="created_at" type="string | null">When the message was stored.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="items" type="object[]">
  The line items, ordered by `number`. Each row merges the quotation snapshot with its catalog match and carries its own suggested alternatives.

  <Expandable title="item">
    <ResponseField name="id" type="string">The `quotation_items` row id.</ResponseField>
    <ResponseField name="item_id" type="string | null">The matched catalog `items` id, or `null` if unmatched.</ResponseField>
    <ResponseField name="number" type="integer">1-based position of the line on the quotation.</ResponseField>
    <ResponseField name="sku" type="string | null">Product SKU.</ResponseField>
    <ResponseField name="brand" type="string | null">Product brand.</ResponseField>
    <ResponseField name="description" type="string">Product description.</ResponseField>
    <ResponseField name="eta" type="integer | null">Lead time in days.</ResponseField>
    <ResponseField name="cost" type="null">Always `null` — the catalog carries no cost column yet.</ResponseField>
    <ResponseField name="price" type="number | null">Unit price.</ResponseField>
    <ResponseField name="margin" type="number | null">Margin, read from the catalog's `margin_pct`.</ResponseField>
    <ResponseField name="quantity" type="integer | null">Quantity requested.</ResponseField>
    <ResponseField name="stock" type="number | null">Available stock, read from the catalog.</ResponseField>
    <ResponseField name="total" type="number | null">Line total.</ResponseField>
    <ResponseField name="overridden" type="boolean">Always `false` from this endpoint.</ResponseField>
    <ResponseField name="suggested_items" type="object[]">
      Runner-up catalog matches for this line — informational alternatives shown in the editor.

      <Expandable title="suggested_item">
        <ResponseField name="item_id" type="string">The catalog `items` id.</ResponseField>
        <ResponseField name="sku" type="string | null">Alternative SKU.</ResponseField>
        <ResponseField name="brand" type="string | null">Alternative brand.</ResponseField>
        <ResponseField name="description" type="string | null">Alternative description.</ResponseField>
        <ResponseField name="cost" type="null">Always `null`.</ResponseField>
        <ResponseField name="price" type="number | null">Alternative unit price.</ResponseField>
        <ResponseField name="stock" type="number | null">Alternative stock.</ResponseField>
        <ResponseField name="score" type="number | null">Match score.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  Line-item fields are **merged**: the `quotation_items` snapshot wins, falling back to the live catalog (`items`) row for `sku`, `brand`, `description`, `eta`, and `price`. `margin` and `stock` always come from the catalog, and `cost` is always `null` (there is no catalog cost column yet). This keeps historical prices intact while still surfacing fresh catalog data where the snapshot is empty.
</Note>

## Errors

<ResponseField name="401" type="Invalid or expired token">
  The JWT is missing, malformed, or expired.
</ResponseField>

<ResponseField name="404" type="Quotation not found: {id}">
  No quotation with that id belongs to the caller's company.
</ResponseField>

<ResponseField name="404" type="No company linked to this user">
  The authenticated user has no company.
</ResponseField>
